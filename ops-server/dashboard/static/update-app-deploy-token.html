Updating the Enablement App — creating the deploy token

When a newer version of the Dynatrace Enablements app is available, the Administration → App version page shows an Update now button.

  • If your tenant is managed by the enablement team (the central enablement Dynatrace account), leave the token field blank and click Update now — the update is applied for you, no token needed.
  • If this is your own Dynatrace tenant, you provide a one-time platform token so the update can install the new app version into your environment. This page explains how to create it.

The token is used once to install the update and is then discarded. It is never stored in the app.

1. Create a platform token in your tenant

  1. In your Dynatrace environment, open Platform tokens (search for “Platform tokens” in the menu, or go to Settings → Platform tokens / Account Management → Identity & access management → Platform tokens).
  2. Choose Generate new token.
  3. Name it, for example enablement-app-update.
  4. Optionally set a short expiration (a day is plenty — it is only used once).
  5. Add these permissions (scopes):
ScopeWhy it is needed
app-engine:apps:installinstall the new app version
app-engine:apps:runactivate the installed version
app-engine:apps:deletereplace the previous version
settings:objects:readread the environment settings the install checks
settings:objects:writeapply the app’s post-install settings (e.g. network allow-list)
  1. Generate the token and copy it. It looks like dt0s16.XXXXXXXX… You can only copy it once.
You need permission to create tokens and to install apps in your tenant. If the button later reports a missing permission, your token is missing one of the scopes above — recreate it with all five.

2. Paste it into the app

  1. In the Enablement app, go to Administration → App version.
  2. Click Check for updates. If a newer version exists you will see Update available.
  3. Paste the token into Platform token (only for your own tenant).
  4. Click Update now. A new browser tab opens and runs the install (about 1–2 minutes).
  5. When it reports App upgraded, return to the app and reload the page to load the new version.

3. Clean up (optional)

The token is discarded by the updater after a single use. If you set no expiration, you can delete it afterwards from the same Platform tokens page.

What happens to the token (security)

  • It is sent over HTTPS to Autonomous Enablement by Dynatrace (Orbital), held encrypted for at most a few minutes, used once to install the update, and then deleted.
  • It is never written to the app, to logs, or to a URL.
  • It only authorises actions in your tenant — the token you create carries your own account’s permissions, nothing more.

If you would rather not handle a token, ask the enablement team to deploy the update for you from Autonomous Enablement (Orbital) — the Register / deploy via Orbital link next to the button.
